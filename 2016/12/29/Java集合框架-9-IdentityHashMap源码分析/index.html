简介

IdentityHashMap在平时并不常用，通过例子来简单感受下。

IdentityHashMap和HashMap没有任何关系，它俩使用的数据结构完全不同。

说明：IdentityHashMap只有在key是同一个引用时才会覆盖，而HashMap则不会。

IdentityHashMap using reference-equality in place of object-equality when comparing keys (and values)

  • reference-equality: k1 == k2
  • object-equality: k1==null ? k2==null : k1.equals(k2)
  1. IdentityHashMap的key和value都可以为null
  2. 无序的
  3. 非线程安全的

数据结构

IdentityHashMap的数据结构很简单，底层就是一个Object数组，在逻辑上可以看成是一个环形的数组， 没有节点类型，数组的偶数位存放key，奇数位存放value。

容量为16的哈希表实际申请数组大小为32，当元素的个数(11)大于数组大小(32)的三分之一时扩容。

解决冲突的办法是开放定址法：也就是根据计算得到散列的位置索引，如果发现该位置上已有元素，则往后查找，直到找到空位置。当元素个数达到一定阈值时，Object数组会进行扩容。
